# 模板管理功能使用指南

## 📁 功能概述

模板管理功能允许您保存、加载、导出和导入系统架构设计，方便您：
- 💾 保存优秀的架构设计为模板
- 📥 快速加载已有架构
- 🔄 在不同项目间复用架构
- 📊 管理多个架构方案

---

## 🎯 核心功能

### 1. 保存为模板
**用途**: 将当前架构保存为可复用的模板

**操作步骤**:
1. 设计好您的系统架构
2. 点击顶部工具栏的"📁 模板管理"
3. 点击"💾 保存为模板"按钮
4. 填写模板信息：
   - **模板名称** (必填): 例如"电商系统架构"
   - **描述**: 描述模板的用途和特点
   - **分类**: 选择最接近的分类
   - **标签**: 用逗号分隔的关键词
5. 点击"保存"

**保存的信息**:
- ✅ 所有节点及其配置
- ✅ 所有连接关系
- ✅ 节点位置信息
- ✅ 元数据（名称、描述、分类、标签）
- ✅ 创建时间和版本号

---

### 2. 加载模板
**用途**: 从模板库中加载架构到画布

**操作步骤**:
1. 点击"📁 模板管理"
2. 在模板列表中找到目标模板
3. 点击"加载"按钮
4. 确认加载（会替换当前画布内容）

**注意事项**:
- ⚠️ 加载模板会**替换**当前画布内容
- ⚠️ 如有未保存的内容，请先保存
- ✅ 可以在加载前导出当前架构作为备份

---

### 3. 导出模板
**用途**: 将模板导出为 JSON 文件，便于分享和备份

**操作步骤**:
1. 点击"📁 模板管理"
2. 找到要导出的模板
3. 点击"导出"按钮
4. 选择保存位置
5. 获得 `.json` 格式的模板文件

**导出文件内容**:
```json
{
  "metadata": {
    "id": "template-1709251234567",
    "name": "电商系统架构",
    "description": "完整的电商系统架构设计",
    "category": "ecommerce",
    "tags": ["微服务", "MySQL", "Redis"],
    "version": 1,
    "createdAt": "2026-03-01T02:00:00.000Z",
    "updatedAt": "2026-03-01T02:00:00.000Z"
  },
  "content": {
    "nodes": [...],
    "edges": [...]
  }
}
```

---

### 4. 导入模板
**用途**: 从 JSON 文件导入模板到模板库

**操作步骤**:
1. 点击"📁 模板管理"
2. 点击"📥 导入模板"区域
3. 选择 `.json` 格式的模板文件
4. 等待导入完成
5. 在模板列表中查看新导入的模板

**导入来源**:
- ✅ 自己导出的模板文件
- ✅ 团队分享的模板
- ✅ 官方提供的最佳实践模板

---

### 5. 保存当前项目
**用途**: 快速保存当前进度，防止丢失

**操作步骤**:
1. 点击"📁 模板管理"
2. 点击"💿 保存当前项目"
3. 系统自动保存到浏览器的 LocalStorage

**特点**:
- ✅ 快速保存（一键完成）
- ✅ 自动覆盖旧版本
- ✅ 下次打开浏览器时自动恢复
- ⚠️ 只在当前浏览器有效

---

### 6. 删除模板
**用途**: 清理不再需要的模板

**操作步骤**:
1. 点击"📁 模板管理"
2. 找到要删除的模板
3. 点击"删除"按钮
4. 确认删除

**注意事项**:
- ⚠️ 删除后无法恢复
- ✅ 建议先导出备份再删除

## 💡 使用场景

### 场景 1: 保存优秀设计
```
1. 完成一个电商系统架构设计
2. 点击"模板管理" → "保存为模板"
3. 填写信息：
   - 名称：电商系统架构
   - 分类：电商系统
   - 标签：微服务，MySQL, Redis, 订单，支付
4. 保存后，下次可以快速加载使用
```

### 场景 2: 团队分享
```
1. 导出自己的架构模板为 JSON
2. 分享给团队成员
3. 团队成员导入模板
4. 基于模板继续开发
```

### 场景 3: 方案对比
```
1. 设计方案 A，保存为"方案 A-微服务"
2. 修改架构，设计方案 B，保存为"方案 B-单体"
3. 在两个模板间切换对比
4. 选择最优方案
```

### 场景 4: 版本管理
```
1. 完成 V1 版本，保存为"系统架构 v1.0"
2. 迭代更新，保存为"系统架构 v2.0"
3. 保留历史版本，便于回溯
```

## 🔧 技术细节

### 存储机制
- **存储位置**: 浏览器 LocalStorage
- **存储键**: `meta-arch-templates`
- **数据格式**: JSON
- **容量限制**: 约 5-10MB（取决于浏览器）

### 数据存储结构
```typescript
interface ArchitectureTemplate {
  metadata: {
    id: string              // 唯一标识
    name: string            // 名称
    description: string     // 描述
    category: string        // 分类
    tags: string[]          // 标签
    version: number         // 版本号
    createdAt: Date         // 创建时间
    updatedAt: Date         // 更新时间
  }
  content: {
    nodes: CustomNode[]     // 节点数组
    edges: Edge[]          // 连接数组
  }
}
```

### 自动功能
- ✅ 自动生成唯一 ID
- ✅ 自动记录时间戳
- ✅ 自动递增版本号
- ✅ 自动保存当前项目（手动触发）

## ⚠️ 注意事项

### 数据安全
- ⚠️ 数据存储在**浏览器本地**
- ⚠️ 清除浏览器缓存会丢失数据
- ✅ 建议定期导出重要模板备份
- ✅ 不要依赖单一存储方式

### 兼容性
- ✅ 支持所有现代浏览器
- ⚠️ 不同浏览器间数据不共享
- ⚠️ 清除缓存后数据丢失

### 性能
- ✅ 支持存储多个模板
- ⚠️ 建议保留 10-20 个模板以内
- ⚠️ 过大的模板可能影响性能

## ❓ 常见问题

### Q1: 模板保存在哪里？
**A**: 保存在浏览器的 LocalStorage 中，位置：
```
浏览器 → 开发者工具 → Application → Local Storage
键名：meta-arch-templates
```

### Q2: 清除缓存后模板会丢失吗？
**A**: 是的！清除浏览器缓存会删除所有模板。建议：
- ✅ 定期导出重要模板
- ✅ 使用浏览器的"不清除此站点数据"功能

### Q3: 可以在不同浏览器间同步模板吗？
**A**: 当前版本不支持。可以通过导出/导入功能手动迁移：
1. 在浏览器 A 导出模板
2. 保存 JSON 文件
3. 在浏览器 B 导入 JSON 文件

### Q4: 模板有数量限制吗？
**A**: 理论上无限制，但受 LocalStorage 容量限制（约 5-10MB）。建议：
- ✅ 保留 10-20 个常用模板
- ✅ 删除不需要的模板
- ✅ 导出备份后清理

### Q5: 可以修改已保存的模板吗？
**A**: 当前版本不支持直接修改。可以：
1. 加载模板到画布
2. 修改架构
3. 保存为新模板（或覆盖旧版本）
