# 🎨 模块与画布集成使用指南

## 📖 概述

模块管理器现在已与画布完全集成！您可以：
- ✅ 将画布中的节点关联到模块
- ✅ 在模块面板中查看和管理关联的节点
- ✅ 从模块中移除节点
- ✅ 查看哪些节点还未分配给模块

---

## 🚀 使用步骤

### 步骤 1: 创建架构节点

1. 在画布上添加节点（前端、API、服务等）
2. 配置节点的属性和样式
3. 建立节点之间的连接关系

### 步骤 2: 创建模块

1. 点击顶部工具栏的 **"🧩 模块管理"** 按钮
2. 点击 **"➕ 创建新模块"**
3. 填写模块信息并创建

### 步骤 3: 将节点关联到模块

**方法一：在模块面板中关联**
1. 打开模块管理面板
2. 点击一个模块查看详情
3. 在"添加节点"区域找到未分配的节点
4. 点击节点旁边的 **"+ 添加"** 按钮

**方法二：右键菜单（后续版本）**
- 右键点击画布中的节点
- 选择"添加到模块"
- 选择目标模块

### 步骤 4: 查看和管理关联

1. 点击模块查看详情
2. 在"关联节点"区域可以看到所有已关联的节点
3. 点击 **"✕ 移除"** 可以将节点从模块中移除

## 🔍 常见问题

### Q1: 一个节点可以关联到多个模块吗？

**A**: 当前版本**不支持**。一个节点只能关联到一个模块。这是为了保持清晰的模块边界。

### Q2: 删除模块后，关联的节点会怎样？

**A**: 删除模块不会删除节点。节点会从模块中解绑，继续保留在画布中。

### Q3: 删除节点后，模块中的关联会怎样？

**A**: 系统会自动清理。如果节点被删除，模块中的关联记录也会被移除。

### Q4: 如何查看某个节点属于哪个模块？

**A**: 当前版本需要在模块面板中查看。后续版本会在节点上显示模块标识。

### Q5: 可以不关联节点吗？

**A**: 可以！模块可以不关联任何节点，作为纯逻辑模块存在。

## 💡 最佳实践

### ✅ 推荐做法

1. **按功能分组**: 将相关功能的节点关联到同一个模块
   ```
   UserModule → [UserService, UserAPI, UserFrontend]
   ```

2. **保持清晰边界**: 不同业务领域的节点不要混在同一个模块
   ```
   ✅ UserModule 只包含用户相关节点
   ✅ OrderModule 只包含订单相关节点
   ```

3. **及时更新关联**: 添加新节点后，及时关联到对应模块

4. **合理使用模块**: 
   - 核心业务 → domain 模块
   - 应用服务 → application 模块
   - UI 组件 → presentation 模块

### ❌ 避免做法

1. **随意关联**: 不要把不相关的节点关联到同一个模块
2. **过度拆分**: 一个节点不要拆分到太多模块
3. **忘记关联**: 创建节点后忘记关联到模块
